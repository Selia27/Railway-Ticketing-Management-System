# Railway Ticketing Management System 

## About The Project
This project is a comprehensive database management system designed for a railway ticketing agency. It was developed as an academic project to demonstrate practical skills in relational database architecture and web interface development.

The system handles the full lifecycle of train management, route tracking, and ticket reservations, offering dynamic reporting and automated data validation.

## Built With
* **Database:** Oracle SQL
* **Backend Logic:** PL/SQL
* **Frontend/UI:** Oracle APEX

## Key Features
* **Relational Architecture:** Complex database schema integrating trains, routes, passenger data, and ticketing.
* **Automated Processing:** PL/SQL blocks for automatic primary key generation and form validation.
* **Dynamic Reporting:** Complex SQL queries utilizing multiple `JOIN` operations and aggregation functions to generate financial and traffic load reports.
* **Interactive UI:** Parameterized views and search functionalities using APEX Page Items.

## Files Included
* `TrenSchema.sql`: DDL scripts for table creation and constraints.
* `TrenDate.sql`: DML scripts containing the initial dataset.
* `Documentation.pdf`: Visual presentation of the application's interface and underlying queries.
